# Тестирование генерации отчёта для задания

## Context

Пользователь хочет протестировать генерацию Markdown-отчёта по расшифровке аудио через API. Конкретное задание `cb18a610...` не существует в файловой системе. Выбрано завершённое задание `12e7dfa4-374d-46dc-bd63-f2e395a157ff` (149 сегментов, ~16 КБ текста).

**Проблема:** Функция `load_segments_file()` в `src/models/report.py` ищет только файлы `*_segments.txt`. У задания `12e7dfa4` есть `_segments.json` (формат MLX Whisper JSON) и `*.txt` (полный текст транскрипции), но НЕТ `*_segments.txt`. При вызове API-эндпоинта `POST /api/v1/report/` функция вернёт `None`, и отчёт не будет сгенерирован.

**Решение:** Расширить `load_segments_file()` для поддержки формата `_segments.json`.

## Requirements

- `load_segments_file()` должен находить текст транскрипции из двух источников:
  1. `*_segments.txt` (приоритет, обратная совместимость)
  2. `_segments.json` с извлечением поля `text` из каждого сегмента
- Тестировать полный цикл: вызов API → генерация в фоне → проверка результата

## Design

### Изменение 1: `src/models/report.py`

Функция `load_segments_file(job_path)` (строки 18-49):

**Сейчас:** перебирает `os.listdir()`, ищет файл, заканчивающийся на `_segments.txt`.

**Новое поведение:**
1. Сначала ищет `*_segments.txt` (как раньше) — при нахождении читает и возвращает
2. Если не найден — ищет `*_segments.json`, парсит JSON, извлекает `segments[].text`, объединяет с переносами строк
3. Если не найден ни один — логирует warning и возвращает `None`

Извлекаемый текст из JSON должен быть читаемым человеком — последовательность `segment.text` по порядку.

### Изменение 2: Тестирование через API

```bash
# 1. Запустить генерацию отчёта
curl -X POST http://localhost:8801/api/v1/report/12e7dfa4-374d-46dc-bd63-f2e395a157ff

# 2. Подождать завершения (проверять наличие файла в папке задания)
sleep 30

# 3. Проверить результат
ls data/12e7dfa4-374d-46dc-bd63-f2e395a157ff/*.md
cat data/12e7dfa4-374d-46dc-bd63-f2e395a157ff/Отчет\ по\ заданию\ 12e7dfa4*.md
```

### Изменение 3: Проверка

- Отчёт сгенерирован в `data/12e7dfa4-374d-46dc-bd63-f2e395a157ff/Отчет по заданию 12e7dfa4-374d-46dc-bd63-f2e395a157ff.md`
- Файл содержит Markdown-контент
- API вернул `{"status": "started"}` при вызове
- Логи не содержат ошибок генерации отчёта
