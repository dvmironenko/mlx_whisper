# Отображение размера файла в карточке задания

## Контекст

В карточках заданий на веб-интерфейсе отображаются имена файлов, но не показывается их размер. Пользователь хочет видеть размер каждого файла рядом с именем для быстрой оценки содержимого задания.

## Требования

- В строке файла карточки задания отображать размер рядом с именем
- Формат: автоматический (Б/КБ/МБ/ГБ) — использовать существующую функцию `formatFileSize()`
- Обратная совместимость: фронтенд должен корректно обрабатывать старый формат (массив строк)

## Изменения

### Бэкенд

**src/services/transcription_service.py**

- `get_job()` (строка 119): заменить `[f for f in os.listdir(...)]` на список объектов `{name, size}`
- `list_jobs()` (строки 134-138): аналогично, строить объекты с размером

**src/services/job_manager.py**

- `list_all()` (orphaned handling): строить объекты `{name, size}` для orphaned директорий

### Фронтенд

**src/templates/index.html**

- `createJobCard()` (строки 807-868):
  - Парсить `f` — строка или объект `{name, size}`
  - Рендерить `<span class="file-size">${formatFileSize(size)}</span>` после имени файла
  - Заменить все ссылки на переменную `f` на `fileName` (view, download, delete)

## Повторное использование

- `formatFileSize()` (index.html, строка 615) — уже реализована
- CSS-класс `.file-size` (new_style.css, строка 1271) — уже существует

## Верификация

1. Запустить приложение: `source .venv/bin/activate && python src/main.py`
2. Загрузить файл транскрипции
3. Открыть карточку завершенного задания — увидеть размер файла рядом с именем
4. Проверить разные размеры файлов: маленький (Б), средний (КБ), большой (МБ)
5. Проверить обратную совместимость: создать orphaned директорию с файлом
