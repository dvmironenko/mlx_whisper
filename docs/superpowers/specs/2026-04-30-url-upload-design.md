# Дизайн: Загрузка видео по URL

## Контекст

Проект MLX-Whisper поддерживает транскрипцию аудиофайлов через загрузку. Необходимо добавить возможность загрузки видео/аудио по URL (YouTube, Vimeo, прямые ссылки) для удобства работы с онлайн-контентом.

## Требования

### Функциональные
1. Загрузка видео/аудио по URL
2. Поддержка YouTube, Vimeo и прямых HTTP/HTTPS ссылок
3. Автоматическое определение типа контента
4. Синхронная обработка (без ожидания завершения)

### Нефункциональные
1. Максимальный размер: 2 ГБ
2. Таймаут скачивания: 600 секунд
3. Безопасность: валидация URL
4. Использование существующих паттернов проекта

## Параметры эндпоинта `POST /api/v1/transcribe-url`

| Параметр | Тип | Описание |
|----------|-----|----------|
| `url` | string | URL видео/аудио (обязательный) |
| `language` | string | Язык (опционально) |
| `model` | string | Модель Whisper (опционально) |
| `word_timestamps` | boolean | Временные метки (опционально) |

## Безопасность

1. **Белый список доменов**: `youtube.com`, `youtu.be`, `vimeo.com`, прямые HTTP/HTTPS
2. **Запрещённые схемы**: `file://`, `ftp://`, `gopher://`
3. **Максимальный размер**: 2 ГБ
4. **Таймаут скачивания**: 600 секунд
5. **Валидация пути**: предотвращение directory traversal

## Обработка ошибок

| Код | Описание |
|-----|----------|
| `400` | Невалидный URL или не поддерживаемый домен |
| `413` | Превышен максимальный размер файла |
| `408` | Таймаут скачивания |
| `500` | Ошибка скачивания/транскрипции |

## Порядок выполнения

1. Валидация URL
2. Скачивание через `yt-dlp` во временный файл
3. Проверка размера
4. Конвертация в WAV (16kHz, mono)
5. Транскрипция через MLX-Whisper
6. Сохранение результатов в `data/uploads/{job_id}/`
7. Возврат `job_id` для дальнейшей проверки статуса
