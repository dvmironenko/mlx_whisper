# MLX-Whisper Web Transcription Service

## Обзор приложения

Приложение представляет собой веб-сервис для транскрибации и перевода аудио с использованием оптимизированной модели Whisper от Apple (MLX-Whisper). Оно предоставляет как веб-интерфейс, так и REST API для преобразования речи в текст с поддержкой нескольких языков и гибких параметров обработки.

## Быстрый старт

### Установка зависимостей

```bash
# Создание и активация виртуального окружения
python -m venv .venv
source .venv/bin/activate

# Установка зависимостей
pip install -r src/requirements.txt
```

### Запуск приложения

```bash
# Создать .env файл в корне проекта (см. раздел Конфигурация)
# или использовать значения по умолчанию

# Запуск сервера
python src/main.py

# Сервер будет запущен на http://localhost:8801
```

### Пример использования

```bash
# Проверка состояния сервиса
curl http://localhost:8801/api/v1/health

# Получение конфигурации
curl http://localhost:8801/api/v1/config

# Транскрипция аудио
curl -X POST "http://localhost:8801/api/v1/transcribe" \
  -F "file=@audio.mp3" \
  -F "language=ru"
```

## REST API

### Эндпоинты

| Эндпоинт | Метод | Описание |
|----------|-------|----------|
| `/` | GET | Веб-интерфейс (HTML шаблон `new_index.html`) |
| `/api/v1/health` | GET | Проверка состояния сервиса |
| `/api/v1/config` | GET | Конфигурация из `.env` |
| `/api/v1/models` | GET | Список поддерживаемых моделей |
| `/api/v1/jobs/{job_id}` | GET | Статус задачи транскрипции |
| `/api/v1/transcribe` | POST | Транскрипция аудиофайлов |

### Примеры использования

#### 1. Проверка состояния сервиса

```bash
curl http://localhost:8801/api/v1/health
# Ответ: {"status": "healthy", "version": "1.0.0"}
```

#### 2. Получение конфигурации

```bash
curl http://localhost:8801/api/v1/config
# Ответ:
# {
#   "initial_prompt": "",
#   "remove_silence": true,
#   "silence_threshold": -60.0,
#   "silence_duration": 0.5,
#   "language": "ru",
#   "no_speech_threshold": 0.4,
#   "hallucination_silence_threshold": 0.8
# }
```

#### 3. Получение списка моделей

```bash
curl http://localhost:8801/api/v1/models
# Ответ: {"supported_models": ["tiny", "base", "small", "medium", "turbo", "large"]}
```

#### 4. Транскрипция аудио

```bash
curl -X POST "http://localhost:8801/api/v1/transcribe" \
  -F "file=@audio.mp3" \
  -F "language=ru" \
  -F "model=turbo" \
  -F "word_timestamps=true"
```

### Параметры эндпоинта `/api/v1/transcribe`

**Обязательные параметры:**

- `file` - Аудиофайл для обработки (WAV, MP3, M4A, FLAC, AAC, OGG, WMA, WEBM, MP4)

**Опциональные параметры:**

| Параметр | Тип | Описание | Значение по умолчанию |
|----------|-----|----------|----------------------|
| `language` | string | Язык аудио (empty = auto-detect) | значение из `.env` |
| `task` | string | Тип задачи: transcribe или translate | значение из `.env` |
| `model` | string | Размер модели: tiny, base, small, medium, turbo, large | значение из `.env` (`large`) |
| `word_timestamps` | boolean | Включение временных меток для слов | значение из `.env` (false) |
| `condition_on_previous_text` | boolean | Использование предыдущего текста для контекста | значение из `.env` (true) |
| `no_speech_threshold` | float | Порог отсутствия речи (0.0-1.0) | значение из `.env` (0.4) |
| `hallucination_silence_threshold` | float | Порог галлюцинаций/тишины (0.0-1.0) | значение из `.env` (0.8) |
| `initial_prompt` | string | Начальный текст для контекстной транскрипции | значение из `.env` |
| `remove_silence` | boolean | Удаление тишины в начале и конце аудио | значение из `.env` (true) |
| `silence_threshold` | float | Порог для обнаружения тишины в dB | значение из `.env` (-60.0) |
| `silence_duration` | float | Минимальная длительность тишины для удаления в секундах | значение из `.env` (0.5) |

**Примечание:** Все параметры опциональны. Если не указаны, используются значения из `.env` файла.

## Веб-интерфейс

### Основные возможности

- **Drag & Drop** загрузка аудиофайлов с визуальной обратной связью
- **Тёмная/светлая тема** с переключателем в правом верхнем углу
- **Аккордеон** для сворачивания/разворачивания параметров транскрипции
- **Динамические кнопки действий**: Копировать, Основной текст, Сегменты (только при наличии timestamps)
- Автоматическая загрузка конфигурации из `.env` при открытии страницы
- **Поддержка языков**: русский, английский, немецкий, французский, испанский

### Дизайн и темы

**Светлая тема (по умолчанию):**

- Primary color: #2563eb (синий)
- Secondary color: #7c3aed (фиолетовый)
- Background dark: #f8fafc
- Card background: #ffffff

**Тёмная тема (dark):**

- Primary color: #60a5fa (голубой)
- Secondary color: #a78bfa (светло-фиолетовый)
- Background dark: #0f172a
- Card background: #1e293b

**Theme Switcher Design:**

- **Minimal gap** переключателя (кнопка тесно прижата к лейблу)
- **Смещённый значок** (thumb сдвинут на -4px для визуального выравнивания)
- Позиционирование: правый верхний угол header'а (absolute positioning)

### Стили и UI

- Стеклянный эффект (glassmorphism) для карточек
- Градиентные элементы и анимации
- Адаптивный дизайн для мобильных устройств (Media queries: 768px, 480px)
- Кастомные скроллбары с тематической цветовой палитрой

## Конфигурация

### Параметры сервера

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `MLX_WHISPER_HOST` | Хост сервера | 0.0.0.0 |
| `MLX_WHISPER_PORT` | Порт сервера | 8801 |
| `MLX_WHISPER_DEBUG` | Режим отладки | false |
| `MAX_FILE_SIZE_MB` | Максимальный размер файла в МБ | 500 |
| `CHUNK_SIZE_KB` | Размер chunk для чтения файлов в КБ | 8 |
| `CONVERSION_TIMEOUT` | Таймаут конвертации в секундах | 600 |
| `TRANSCRIPTION_TIMEOUT` | Таймаут транскрипции в секундах | 3600 |
| `MODELS_DIR` | Путь к каталогу моделей | models |
| `RESULTS_DIR` | Путь к каталогу результатов | results |
| `RESULTS_RETENTION_DAYS` | Срок хранения результатов в днях | 30 |
| `LOGS_DIR` | Путь к каталогу логов | logs |
| `LOG_LEVEL` | Уровень логирования | INFO |

### Параметры аутентификации

| Переменная | Описание |
|------------|----------|
| `MLX_WHISPER_API_KEY` | API ключ для аутентификации (оставить пустым для отключения) |

### Параметры обработки аудио

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `REMOVE_SILENCE` | Удалять тишину в начале и конце аудио | true |
| `SILENCE_THRESHOLD` | Порог для обнаружения тишины в dB | -60.0 |
| `SILENCE_DURATION` | Минимальная длительность тишины для удаления в секундах | 0.5 |

### Параметры транскрипции

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `LANGUAGE` | Язык по умолчанию для транскрипции (empty = auto-detect) | ru |
| `TASK` | Тип задачи по умолчанию: transcribe или translate | transcribe |
| `MODEL` | Модель по умолчанию: tiny, base, small, medium, turbo, large | large |
| `WORD_TIMESTAMPS` | Включить word-level timestamps | false |
| `CONDITION_ON_PREVIOUS` | Использовать предыдущий текст для контекста | true |
| `NO_SPEECH_THRESHOLD` | Порог отсутствия речи (0.0-1.0) | 0.4 |
| `HALLUCINATION_SILENCE_THRESHOLD` | Порог галлюцинаций/тишины (0.0-1.0) | 0.8 |
| `INITIAL_PROMPT` | Начальный текст для контекстной транскрипции (опционально) | - |

### Конфигурация логирования

Конфигурация логирования в `config.py` с ротацией файлов:

- `app.log` - основной файл логов (10MB, 5 резервных копий)
- `error.log` - файл ошибок (10MB, 5 резервных копий)
- Логи выводятся в stdout и файлы одновременно

### Пример .env файла

```bash
# ========================================
# Server Settings - Настройки сервера
# ========================================

MLX_WHISPER_HOST=0.0.0.0
MLX_WHISPER_PORT=8801
MLX_WHISPER_DEBUG=false

# ========================================
# Audio Processing - Обработка аудио
# ========================================

MAX_FILE_SIZE_MB=500
CHUNK_SIZE_KB=8

# ========================================
# Timeouts - Таймауты
# ========================================

CONVERSION_TIMEOUT=600
TRANSCRIPTION_TIMEOUT=3600

# ========================================
# Directories - Каталоги
# ========================================

MODELS_DIR=models
RESULTS_DIR=results
RESULTS_RETENTION_DAYS=30
LOGS_DIR=logs
LOG_LEVEL=INFO

# ========================================
# Authentication - Аутентификация
# ========================================

MLX_WHISPER_API_KEY=

# ========================================
# Silence Removal - Удаление тишины
# ========================================

REMOVE_SILENCE=true
SILENCE_THRESHOLD=-60.0
SILENCE_DURATION=0.5

# ========================================
# Language - Язык
# ========================================

LANGUAGE=ru

# ========================================
# Transcription Thresholds - Пороги транскрипции
# ========================================

NO_SPEECH_THRESHOLD=0.6
HALLUCINATION_SILENCE_THRESHOLD=0.8

# ========================================
# Initial Prompt - Начальный промпт
# ========================================

INITIAL_PROMPT=

# ========================================
# Transcription Settings - Настройки транскрипции
# ========================================

TASK=transcribe
MODEL=turbo
WORD_TIMESTAMPS=false
CONDITION_ON_PREVIOUS=true
```

│       └── files.py              # Работа с файлами

├── tests/

### Ключевые модули

#### `src/config.py` - Конфигурация приложения

- Загрузка переменных из `.env` файла
- Настройка логирования с ротацией файлов
- Динамическая конфигурация параметров транскрипции

#### `src/models/transcription.py` - Бизнес-логика

- `transcribe_audio()` - выполнение транскрипции с помощью MLX-Whisper
- `convert_numpy_types()` - конвертация numpy типов для JSON
- `sanitize_result()` - очистка данных (NaN, Infinity) для JSON

#### `src/utils/audio.py` - Работа с аудио

- `convert_to_wav()` - конвертация в WAV (16kHz, mono) через FFmpeg
- `validate_audio_file()` - валидация аудиофайла
- `get_audio_duration()` - получение длительности

#### `src/utils/files.py` - Работа с файлами

- `generate_unique_filename()` - генерация уникального имени
- `validate_file_size()` - проверка размера файла
- `validate_file_extension()` - проверка расширения
- `chunked_read()` - потоковое чтение файла
- `delete_file()` - безопасное удаление файла
- `cleanup_old_files()` - очистка старых файлов

#### `src/api/router.py` - API роуты

- `POST /api/v1/transcribe` - транскрипция аудио
- `GET /api/v1/health` - проверка состояния
- `GET /api/v1/models` - список моделей
- `GET /api/v1/config` - конфигурация из `.env`
- `GET /api/v1/jobs/{job_id}` - статус задачи

#### `src/api/dependencies.py` - Зависимости API

- `verify_api_key()` - проверка API ключа
- `get_current_api_key()` - получение текущего ключа

### Поддерживаемые аудиоформаты

WAV, MP3, M4A, FLAC, AAC, OGG, WMA, WEBM, MP4

### Поддерживаемые модели Whisper

- `tiny` - маленькая модель
- `base` - базовая модель
- `small` - малая модель
- `medium` - средняя модель
- `turbo` - турбо модель
- `large` - большая модель

### Оптимизация производительности

- Использование `ThreadPoolExecutor` для параллельной обработки задач
- Частичная обработка файлов (8 КБ чанки) для экономии памяти
- Автоматическая очистка временных файлов после обработки
- Потоковая конвертация аудио через FFmpeg

### Безопасность и надежность

- Полная обработка ошибок с соответствующими HTTP-статусами
- Безопасное удаление временных файлов при ошибках
- JSON сериализация с особым подходом к специальным значениям (NaN, Infinity)
- Опциональная аутентификация через API ключ
- Ротация логов (10MB, 5 резервных копий)

## Заключение

Приложение предоставляет надежное, масштабируемое решение для транскрипции аудио с интуитивно понятным веб-интерфейсом и правильной обработкой ошибок. Использование MLX-Whisper обеспечивает оптимизированные характеристики производительности от Apple, при этом сохраняя доступность через веб-интерфейс.
